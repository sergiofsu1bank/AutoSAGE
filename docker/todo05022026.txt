Checkpoint de Continuidade – AutoSAGE Monitor

Estado atual da plataforma:

Toda a infraestrutura está operacional via Docker Compose, com os seguintes serviços ativos: ORC, Metrics, DCP, EDA Explore, EDA Prepare, ML, LLM, LLM Platform, Monitor e Postgres (pgvector). Todos os containers estão rodando corretamente e as APIs FastAPI estão acessíveis via Swagger.

O módulo AutoSAGE Monitor já foi implementado com arquitetura limpa seguindo o padrão da plataforma:

Router → Service → Repository → Connector → PostgreSQL

Endpoints implementados até o momento:

1. GET /datasets/{dataset_name}/latest
   Retorna o último run executado para um dataset específico.

Dados reconstruídos a partir das tabelas:

* pipeline_metrics
* dcp_monitor_event
* ml_experiment_result

Campos retornados:

* dataset_name
* pipeline_version
* rows_count
* columns_count
* model_name
* metric_name
* metric_value
* duration_ms

2. GET /datasets/{dataset_name}/runs
   Retorna histórico das últimas execuções da pipeline para o dataset.

Campos retornados:

* pipeline_version
* model_name
* metric_name
* metric_value
* duration_ms

Objetivo:

DCP → EDA Explore → EDA Prepare → ML

Usando as tabelas:

* dcp_monitor_event
