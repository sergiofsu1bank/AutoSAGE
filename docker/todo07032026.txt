# AutoSAGE Monitor — Checkpoint de Desenvolvimento

Data: 2026-03-07

## 1. Objetivo do Monitor

O **AutoSAGE Monitor** é o serviço responsável por fornecer **observabilidade da plataforma AutoSAGE**, incluindo:

* saúde da plataforma
* execução de pipelines
* histórico de modelos
* métricas de execução
* timeline de processamento de datasets

Ele funciona como **camada de inspeção operacional da plataforma**.

Arquiteturalmente ele **não executa pipelines**, apenas **consulta e consolida informações** vindas de:

* banco Postgres
* serviços da plataforma

# 4. Endpoints de Dataset

## 4.1 Pipeline Execution

Endpoint:

```
GET /datasets/{dataset_name}/pipeline
```

Retorna:

* versões de pipeline
* modelo treinado
* métrica utilizada
* valor da métrica
* tempo de execução

Query principal:

```
SELECT
    pm.pipeline_version,
    ml.model_name,
    ml.metric_name,
    ml.metric_value,
    pm.duration_ms
FROM pipeline_metrics pm
LEFT JOIN ml_experiment_result ml
    ON pm.trace_id = ml.trace_id
WHERE pm.dataset_name = %s
ORDER BY pm.pipeline_version DESC
LIMIT 20
```

## 4.2 Runs de Dataset


## 4.3 Timeline do Pipeline

Endpoint:

```
GET /datasets/{dataset_name}/timeline
```

Objetivo:

Mostrar **linha temporal da execução do pipeline**, incluindo:

* ingestão
* eda explore
* eda prepare
* ml
* métricas

Cada etapa deve conter:

```
stage
created_at
duration
status
```

Implementação parcialmente iniciada.
